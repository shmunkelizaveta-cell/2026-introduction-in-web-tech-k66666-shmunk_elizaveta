Ход работы: 
Для начала работы была создана папка prometheus. 
В конфигурации были указаны два источника: сам Prometheus и сервис Node Exporter, который используется для сбора системных метрик.
Запуск Node Exporter:
был запущен контейнер Node Exporter. Контейнер был запущен с помощью Docker.
После запуска была выполнена проверка работы сервиса с помощью запроса:
curl http://localhost:9100/metrics
В ответ был получен список метрик, что подтвердило корректную работу Node Exporter.
Запуск Prometheus: 
Был создан Docker-том prometheus-data. Затем была создана отдельная сеть Docker monitoring, чтобы Prometheus и другие сервисы могли взаимодействовать друг с другом.
После этого был запущен Prometheus.
Работа сервиса была проверена через браузер по адресу:
http://localhost:9090
После открытия интерфейса Prometheus можно было убедиться, что система успешно собирает метрики.
Запуск Grafana:
Для хранения данных Grafana был создан отдельный Docker-том grafana-data. Затем контейнер Grafana был запущен в той же сети monitoring, что и Prometheus.
После запуска была выполнена проверка работы сервиса через браузер по адресу:
http://localhost:3000
После входа в Grafana был добавлен новый источник данных. В качестве источника был выбран Prometheus
Создание дашборда:
Далее был создан новый дашборд для отображения метрик системы. В качестве источника данных был выбран Prometheus.
На дашборд была добавлена визуализация метрики:
node_cpu_seconds_total
После этого были добавлены  графики.
Тестирование системы мониторинга:
Сначала был выполнен просмотр запущенных контейнеров:
docker ps
После этого были проверены интерфейсы Prometheus и Grafana. В Prometheus можно было увидеть, что метрики успешно собираются, а в Grafana отображались графики на созданном дашборде.
