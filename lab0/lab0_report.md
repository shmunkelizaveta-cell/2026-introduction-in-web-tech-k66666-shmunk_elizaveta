В ходе выполнения лабораторной работы я использовала терминал macOS для работы с системой контроля версий Git и платформой GitHub.
Для работы с репозиториями я настроила новый SSH-ключ для безопасного подключения.
На платформе GitHub был создан новый репозиторий с названием: https://github.com/shmunkelizaveta-cell/devops-lab-shmunk
Был создан файл README.md, в котором размещено описание проекта, контактная информация и краткий план изучения DevOps.
После этого был добавлен файл .gitignore, содержащий стандартные исключения для операционной системы macOS. Файл был создан и отредактирован через терминал.
Для разработки была создана отдельная ветка develop, и работа над проектом продолжилась уже в этой ветке.
Далее был создан файл CONTRIBUTING.md.
После добавления файлов в репозиторий был выполнен первый коммит с сообщением "Initial project setup", фиксирующий начальную структуру проекта.
Все изменения были отправлены в удалённый репозиторий на GitHub с помощью команды git push, что позволило синхронизировать локальную и удалённую версии проекта.
Затем на GitHub был создан Pull Request из ветки develop в ветку main, содержащий описание внесённых изменений.
После проверки изменений Pull Request был объединён с основной веткой main.
В завершение работы ветка develop была удалена, так как её изменения уже были интегрированы в основную ветку проекта.
