Ход работы
Установка Docker: 
Для выполнения лабораторной работы был установлен Docker Desktop на macOS. После установки была проверена корректность работы программы с помощью команды docker --version. Затем для проверки запуска контейнеров был выполнен тестовый контейнер hello-world. Успешный запуск подтвердил, что Docker установлен и работает корректно.
Работа с готовыми образами:
Далее был загружен официальный образ Ubuntu из Docker Hub с помощью команды docker pull ubuntu:latest. После загрузки образ был запущен в интерактивном режиме (docker run -it ubuntu bash). Внутри контейнера был установлен пакет curl, после чего проверена его версия, чтобы убедиться в корректной установке.
Запуск веб-сервера nginx:
Следующим шагом был запущен контейнер с веб-сервером nginx с пробросом порта на локальную машину (docker run -d -p 8080:80 --name web-server nginx:alpine). После запуска работа сервера была проверена в браузере по адресу http://localhost:8080. Также были просмотрены логи контейнера для проверки его работы.
Управление контейнерами:
Сначала был выполнен просмотр запущенных контейнеров (docker ps) и всех контейнеров (docker ps -a). Затем контейнер был остановлен, снова запущен и удалён. После этого также был удалён образ nginx, чтобы проверить корректность выполнения команд управления контейнерами.
Работа с томами (volumes):
Для изучения механизма хранения данных в Docker был создан том my-volume. Далее был запущен контейнер Ubuntu с подключённым томом (-v my-volume:/data). Внутри контейнера был создан текстовый файл в директории /data. После этого содержимое файла было проверено с помощью команды cat /data/test.txt
